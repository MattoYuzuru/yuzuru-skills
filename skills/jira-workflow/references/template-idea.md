# Description Template — Idea (LYLT)

An Idea description is strictly structured: 9 panels in Jira wiki markup using `{panel}` macros.
Leave unfilled sections with placeholder text — do not delete panels.

```
{panel:title=1. Problem you're solving|borderColor=#cccccc|titleBGColor=#ffed93}
<2-3 sentences on the essence of the request>
{panel}
{panel:title=2. Goal|borderColor=#cccccc|titleBGColor=#ffed93}
 # <Goal 1>
 # <Goal 2>
{panel}
{panel:title=3. Process AS IS|borderColor=#cccccc|titleBGColor=#ffed93}
<How the process works today — 2-3 sentences>
{panel}
{panel:title=4. Process TO BE|borderColor=#cccccc|titleBGColor=#ffed93}
<How the process should work after the change — 2-3 sentences>
{panel}
{panel:title=5. Acceptance criteria|borderColor=#cccccc|titleBGColor=#ffed93}
<How will we confirm the task was done well and solves the problem?>
{panel}
{panel:title=6. Users/Audience|borderColor=#cccccc|titleBGColor=#ffed93}
<Who is the primary user of this change?>
{panel}
{panel:title=7. Design mockups|borderColor=#cccccc|titleBGColor=#ffed93}
<Link to design mockups (may be absent when the idea is first created)>
{panel}
{panel:title=8. Risks/blockers|borderColor=#cccccc|titleBGColor=#ffed93}
<Expected risks/blockers (can be filled in later)>
{panel}
{panel:title=9. Artifacts|borderColor=#cccccc|titleBGColor=#ffed93}
<Links to artifacts that help understand the idea or its context>
{panel}
```
